# TLAH Studio 4.9.8 发布审计

## 本次优化

4.9.8 聚焦 Claude Code 对标中最危险的权限和计划状态缺口，而不是扩大到高风险的多代理重写。

| 优先级 | 问题 | 修复 |
|---|---|---|
| P0 | 完全访问可绕过策略拒绝和安全硬拦截 | 将策略拒绝、安全阻断置于所有权限模式之前；工具执行边界再次校验，覆盖审批后恢复的场景。 |
| P0 | 初始 Plan 状态与恢复检查点不完整，退出计划可能继续保持 Plan | 持久化有效权限和自动批准状态；成功进入/退出时在保存检查点前转换，并重建系统提示。 |
| P0 | 询问用户、退出计划能在自动/完全访问下自行继续 | 用不可绕过的“需要用户交互”元数据统一暂停。 |
| P1 | 退出计划依赖模型写入带未替换占位符的文件 | `exit_plan_mode` 必须传入完整计划，并在已批准后由工具持久化。 |
| P1 | 普通补丁被 UI 一律显示为强制更新 | UI 直接使用签名清单的 `ForceUpdate` 字段。 |
| P1 | 单例签名 URL 会在发布切换时与旧清单错配 | 新清单使用版本化 ECDSA 签名 URL；客户端限制同一 HTTPS 主机，发布仍保留旧 URL 以服务 4.9.7 客户端。 |
| P1 | 发布验证允许“未签名但不受信任”安装包 | `-AllowUntrustedAuthenticode` 仅允许不受信任证书链，绝不允许缺少签名。 |

## 验证

- 300 项 Core xUnit 回归通过（新增计划、完全访问阻断、版本签名 URL 和更新 URL 限制测试）。
- 发布门禁继续执行依赖漏洞审计、Release 构建、签名、安装/启动烟测、清单 ECDSA 校验和服务器原子提升。

## 剩余差距

`LlmService` 仍保留旧循环代码，WinUI 没有独立 UI 自动化项目，且 `AgentRunEngineV2` 仍是顺序单代理执行。多代理编排、消息队列消费和 DAG 调度保留给 5.0，详见 `TLAH_5_0_PHASE2_ORCHESTRATION.md`。
